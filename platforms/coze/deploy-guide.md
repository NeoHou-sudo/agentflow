# ⚡ Coze 扣子 — 部署指南

## 为什么选 Coze？

- 字节跳动出品，稳定可靠
- 云端运行，无需服务器
- 可视化编排，上手快
- 海量插件生态
- 国内访问快（coze.cn）

## 一、注册账号

1. 访问 https://www.coze.cn
2. 使用抖音账号或手机号登录
3. 进入工作台

## 二、创建工作流（通用步骤）

每个工作流的创建流程类似，以下是通用步骤：

### Step 1: 新建工作流
- 点击「创建工作流」→「从空白开始」
- 命名（如"小红书文案批量生成"）

### Step 2: 配置开始节点
- 添加变量 `topic`（字符串类型）
- 描述：笔记主题

### Step 3: 添加大模型节点
- 模型选择：Claude 或豆包
- Prompt：复制本仓库对应工作流的提示词

### Step 4: 配置结束节点
- 输出：大模型节点的结果

### Step 5: 测试并发布
- 点击「试运行」→ 输入参数 → 查看结果
- 确认无误后，点击「发布」→「发布为 Bot」

## 三、获取 API Token

### 个人访问令牌
1. Coze API → 授权 → 个人访问令牌
2. 点击「添加」
3. 填写名称、过期时间、权限范围
4. **复制并保存令牌（只显示一次）**

令牌格式：`Bearer pat_xxxxxxxxxxxx`

### Bot ID
创建 Bot 后，在 Bot 页面 URL 中可找到 Bot ID。

## 四、工作流部署示例

### 小红书文案批量生成
```
节点 1 - 开始节点
  参数名：topic
  类型：字符串
  描述：笔记主题

节点 2 - 大模型节点
  模型：Claude / 豆包
  Prompt：
  你是一个小红书爆款文案专家。
  请根据主题「{{topic}}」，生成10篇不同风格的小红书笔记...

节点 3 - 结束节点
  输出：大模型节点的结果
```

## 五、将 Bot 接入网页/应用

1. 发布 Bot → 选择「Chat SDK」
2. 进入 Playground 调试
3. 获取 `bot_id` 和 `token`
4. 使用 HTML 代码接入（参考 Coze 官方文档）

## 六、常见问题

**Q: Coze 国内版 (coze.cn) 和国际版 (coze.com) 区别？**  
A: 国内版访问更快，插件市场是中文生态；国际版插件更多但访问较慢。

**Q: 工作流测试正常但发布后没反应？**  
A: 检查 Bot 的人设与回复逻辑是否正确配置。

**Q: 如何提高 Bot 回答质量？**  
A: 在"人设与回复逻辑"中详细描述 Bot 的身份、语气、专业领域。
